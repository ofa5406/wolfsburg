# Post-Car Future of Wolfsburg
## Our shared project workspace

A quick tour for the team — what it is, why it helps, and how we use it.

*Parking City → Hub City*

---

# Why we built this

**The problem:** ideas, decisions, numbers and feedback were scattered across chats, files and our heads. When a session ended, knowledge got lost.

**The fix:** one repository that is the **single source of truth** —
the concept, the evidence, the decisions, and a memory so we can pick up
exactly where we left off.

> It's the *workshop* behind the project — not the final boards.

---

# What's inside (the map)

- **`CLAUDE.md`** — project briefing (read first)
- **`HANDOFF.md`** — where we are *right now* (read second)
- **`project/`** — the living description, deliverables, hub typologies
- **`research/`** — 📚 10 cited documents + 199 sources
- **`decisions.md` / `notes.md` / `tasks.md`** — decisions, ideas, to-dos
- **`consultations/`** — tutor feedback, one file each

# Three facts that win the room

1. **Wolfsburg has Germany's highest car density** — 956 cars / 1,000 residents. *"If it works here, it works anywhere."*

2. **Ghent switched 6 sectors in a single weekend for ~€5m** — proof this is fast and affordable, not utopian.

3. **The VW shift wave (~10,000 workers in ~1 hour)** is *predictable, concentrated* demand — the ideal case for shared mobility.

# How we use it — every session

**Start (60-second resume):**
1. Read `CLAUDE.md` → the briefing
2. Read `HANDOFF.md` → what's done / next
3. Need a number or precedent? → `research/README.md`

**End (so nothing is lost):**
1. Write a session log (`sessions/_template.md`)
2. Add a line to `sessions/INDEX.md`
3. Update `HANDOFF.md`
